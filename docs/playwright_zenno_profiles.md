# Настройка Playwright для работы с профилями ZennoPoster

## Основные параметры запуска Chrome

Для успешной работы Playwright с профилями ZennoPoster необходимы следующие настройки:

### Игнорируемые флаги Chrome
```python
ignore_default_args=[
    "--enable-automation",
    "--disable-extensions"
]
```

### Обязательные аргументы запуска
```python
args=[
    f"--profile-directory=Profile {profile_name}",
    "--enable-extensions",
    "--no-sandbox",
    "--disable-web-security",
    "--allow-running-insecure-content"
]
```

## Особенности работы с расширениями

1. Расширения должны быть предварительно установлены в профиль
2. Путь к расширениям должен быть в формате: `{profile_path}/Extensions/{ext_id}/{version}`
3. Необходимо отключить флаг `--disable-extensions` через `ignore_default_args`
4. Добавить флаг `--enable-extensions` в аргументы запуска

## Процесс запуска профиля

1. Закрыть все процессы Chrome перед запуском
2. Проверить наличие профиля и его расширений
3. Запустить Chrome с нужными параметрами
4. Подключиться через CDP к запущенному браузеру
5. Создать новую страницу в существующем контексте

## Важные моменты

- Не использовать `remote-debugging-port` - Playwright сам управляет подключением
- Использовать `headless=False` для отладки
- Проверять наличие всех необходимых расширений перед запуском
- Использовать достаточные таймауты для загрузки страниц и элементов

## Пример конфигурации

```json
{
    "launch_flags": {
        "required": [
            "--user-data-dir={CHROME_DATA_PATH}",
            "--profile-directory=Profile {profile_name}",
            "--no-first-run",
            "--no-default-browser-check",
            "--enable-extensions",
            "--disable-web-security",
            "--allow-running-insecure-content"
        ]
    },
    "timeouts": {
        "page_navigation": 120000,
        "element_search": 60000
    }
}
```

## Проверка работоспособности

Для проверки корректности настроек рекомендуется:
1. Убедиться что профиль открывается
2. Проверить что расширения активны и работают
3. Проверить что можно взаимодействовать с элементами страницы
4. Убедиться что CDP подключение стабильно

## Известные проблемы и решения

1. Если расширения не загружаются:
   - Проверить флаги `--enable-extensions` и отключение `--disable-extensions`
   - Проверить права доступа к папке с расширениями

2. Если не удается подключиться к браузеру:
   - Убедиться что все процессы Chrome закрыты
   - Проверить корректность путей к профилю
   - Увеличить таймауты подключения

3. Если страница загружается без расширений:
   - Проверить структуру папки Extensions в профиле
   - Убедиться что расширения установлены корректно
